# Real tools inventory (v0.4 · Stage R1)

The v0.4 goal: benchmark **real, third-party artifacts**, not only our own interface
generators. This is the reconnaissance — what actually exists, what we can run
**reproducibly** (OSS + our Anthropic key), and what is account-gated (cited, not run).

Toolchain note: this repo is **Python-first** (Node was removed). Prefer `pip`/`uvx`
tools; Node/Docker tools are usable but flagged. Installability claims below are from each
project's docs/PyPI and get **verified for real in R2/R3** (that's the point of the bench).

## 1. OpenAPI → MCP generators (for R2, bucket A)

We feed one real OpenAPI spec through several **real** generators and score each generator's
**actual** tool definitions with `lap score` — neutral "which real generator emits the
leanest agent menu."

| tool | lang / install | notes | in scope? |
| --- | --- | --- | --- |
| **FastMCP** (`fastmcp`) | Python · `pip install fastmcp` | `FastMCP.from_openapi(...)`; already our `mcp_fastmcp` baseline | ✅ have it |
| **openapi-to-mcp** | Python · `pip install openapi-to-mcp` | OpenAPI v2/v3 → MCP; FastAPI integration | ✅ candidate |
| **openapi-mcp** | Python · `pip install openapi-mcp` | zero-config OpenAPI → MCP | ✅ candidate |
| **openapi-mcp-codegen** (cnoe-io) | Python · git/pip | *generates* a Python MCP server package | ✅ candidate |
| **openapi-mcp-generator** | TypeScript · `npx` | standalone TS servers | ⚠️ needs Node |
| **Speakeasy** (Gram / CLI) | hosted / CLI | commercial; free tier unclear | ❌ account — cite only |
| **Stainless** | hosted | commercial | ❌ account — cite only |

**Plan for R2:** verify installability of the ✅ Python candidates, generate from the same
real spec (start with the **live Swagger Petstore**), diff their menus, and add a
`real-generator` comparison next to our synthetic `openapi_full`/`compact_sig`.

## 2. Token optimizers / compressors (for R2/R6, real reductions)

Real tools that shrink an existing MCP surface — measure their **output**, don't reimplement.

| tool | lang / install | what it does | in scope? |
| --- | --- | --- | --- |
| **mcp-compressor** (atlassian-labs) | Python/TS/Rust · `pip install mcp-compressor` | MCP proxy; shrinks tool descriptions 70–97% via progressive disclosure (small surface first, full schema on demand) | ✅ **tested, v0.5 S2** — [`docs/MCP-COMPRESSOR.md`](MCP-COMPRESSOR.md) |
| **LLM-OpenAPI-minifier** (ShelbyJenkins) | Python · git | minifies an OpenAPI's *characters* for prompt-stuffing (LAP optimizes *tokens*) | ✅ candidate (contrast) |
| **token-optimizer-mcp** (ooples) | · git | caching/compression aimed at Claude Code sessions | ⚠️ Claude-Code-specific, less API-menu-general |
| StackOne / Speakeasy runtime | hosted | commercial gateways | ❌ account — cite only |

**Plan:** run `mcp-compressor` on a real MCP server's tool set and score its compressed
surface with `lap score`, head-to-head vs raw and vs our `compact_sig` — a real optimizer vs
ours, on the same input.

## 3. Real MCP servers to score live (for R3, bucket A on real servers)

Score **published** servers' advertised menus with `lap score --mcp-url` — real menus, not
generated by us. The GitHub server is a great heavy real example (~94 tools, ~17.6k tokens).

| server | run it | notes |
| --- | --- | --- |
| **mcp-server-git** | `uvx mcp-server-git` | Python reference server, no Node |
| **mcp-server-fetch** | `uvx mcp-server-fetch` | Python reference server |
| **mcp-server-time** | `uvx mcp-server-time` | Python reference server (small) |
| **filesystem / everything** | `npx` or `docker run mcp/filesystem` | Node or Docker |
| **github-mcp-server** (official) | `docker run mcp/github-official` (needs a GitHub token) | ~94 tools, ~17.6k tokens — the poster child for tool bloat |

**Plan for R3:** start with the `uvx` Python servers (zero Node), add Docker `filesystem` +
`github` if Docker is available; publish a "real MCP servers" section in `LEADERBOARD.md`.

## 4. Real Anthropic features (for R4–R6, the live "gold standard")

Our `tool_search` and `code_exec` are approximations of real Anthropic products. With our
key we can measure the **real** ones head-to-head:

- **Tool Search Tool** (Anthropic, GA'd from the Nov-2025 beta) — server-side lazy tool
  loading; register a large real toolset with it vs naive all-tools → real token + accuracy
  numbers (R5).
- **Code execution with MCP** (Anthropic) — the real code-execution path vs our sandbox
  `code_exec` (R6).
- Cloudflare **Code Mode** — real, but their infra; **cite, don't run** reproducibly.

## What we can and can't claim

- ✅ **Reproducible, neutral:** OSS generators (§1), OSS optimizers (§2), live OSS/reference MCP
  servers (§3), and real Anthropic API features (§4, with our key).
- ❌ **Not reproducible here:** commercial hosted products (Speakeasy Gram, Stainless, StackOne,
  Cloudflare Code Mode) — cited with their published numbers, never passed off as ours.

## Chosen starting set

- **R2 generators:** FastMCP (have) + `openapi-to-mcp` + `openapi-mcp` + cnoe `openapi-mcp-codegen`.
- **R2/R6 optimizer:** `mcp-compressor` (Atlassian).
- **R3 servers:** `mcp-server-git` / `-fetch` / `-time` via `uvx` first; Docker `github`/`filesystem` if available.
- **R4 real API:** live **Swagger Petstore** (public, hosted) as the first non-toy end-to-end target.
